Overview

Purpose

This document describes the Bonus Wheel feature for Crimson Temple, an Asian-mythology themed 5x3 video slot. The feature introduces a high-intensity, visually cinematic wheel mechanic that awards multipliers, fixed coin wins, free spins bundles, and a progressive ‚ÄúDragon Pot‚Äù jackpot tier.

The goal of this feature is to:

  • Provide a clear, reusable specification for engineering, QA, and math teams.
  • Define the exact behaviour of the Bonus Wheel from trigger to payout.
  • Ensure the feature can be certified in regulated markets with minimal rework.
  • Align UI/UX, audio, and thematic elements with the Crimson Temple brand.

Scope

In Scope

  • Bonus Wheel trigger conditions (base game only).
  • Wheel layout, segments, and prize types (coin wins, multipliers, free spins, jackpot tier).
  • Front-end flow and animations from trigger event to result reveal.
  • Back-end state handling, RNG calls, and payout resolution.
  • Integration with game math (RTP contribution, volatility, prize distribution).
  • Basic localisation hooks for labels, tooltips, and messages.

Out of Scope

  • Full base game specification (reels, pay table, line evaluation logic).
  • Back-office configuration tools (campaign overlays, custom wheel mapping per operator).
  • Promotional overlays, seasonal reskins, or time-limited events.
  • Detailed jackpot platform integration (covered in Jackpot Platform spec).

Main Feature Description

The Crimson Temple Bonus Wheel is a single-spin wheel bonus feature with 16 segments. Each segment represents one of four prize classes:

  • Fixed Coin Win (e.g., 20x‚Äì100x total bet)
  • Win Multiplier applied to the spin that triggered the feature
  • Free Spins Bundle with enhanced reel set
  • Dragon Pot ‚Äì entry into a jackpot pick game (Mini / Major / Grand)

Trigger Conditions

  • The Bonus Wheel triggers when 3 or more Wheel Scatter symbols land anywhere on the reels during a base game spin.
  • A secondary trigger path exists via Temple Spirit Meter: every non-winning spin may randomly charge the meter. When the meter reaches 100%, it guarantees a Bonus Wheel trigger on the next spin.
  • Only one Bonus Wheel can be active at a time; feature triggers cannot stack.

High-Level Flow

  1. Base game spin completes and engine evaluates wins.
  2. Engine checks for 3+ Wheel Scatters or a forced trigger via Temple Spirit Meter.
  3. If triggered, reels dim and a modal-style transition animates the player into the Bonus Wheel scene.
  4. Wheel spins for 3‚Äì5 seconds with easing out animation and synchronized soundscape.
  5. RNG resolves final segment; wheel slows and stops on the winning prize.
  6. Prize is granted (coin win, multiplier, free spins, or Dragon Pot entry).
  7. Player is returned to base game or moved into the Free Spins / Dragon Pot bonus.

Prerequisite

Before implementing or enabling the Bonus Wheel feature, the following must be in place:

  • Installed tools: Node.js, build tooling for the FE framework, RNG library dependency.
  • Access rights:
    • Developer access to game server and client code repositories.
    • QA access to staging and certification environments.
    • Math team access to game configuration and simulation tools.
  • A configured and certified RNG source referenced by the game server.
  • Art and sound assets for the wheel (see Assets section).

Dependencies

  • Game Server: Responsible for RNG calls, prize resolution, and balance updates.
  • Jackpot Platform: Required if the Dragon Pot prize is enabled.
  • Base Game Engine: Must surface scatter counts and meter state per spin.
  • Analytics / Telemetry Service: To track feature entry, hit rate, and prize distribution.

The Bonus Wheel feature must not be enabled in production until the base game configuration and math model have been validated by the math team and, where applicable, external labs.

Change Description

Current State: Crimson Temple base game is live with standard free spins feature, but no mid-to-high volatility wheel mechanic.

Target State: Introduce a Bonus Wheel feature that:

  • Triggers in ~1/160 spins (configurable per jurisdiction).
  • Contributes ~25‚Äì30% of total game RTP.
  • Offers a mix of medium coin wins and rare high-value outcomes (Dragon Pot, high multipliers).

Implementation Steps

  1. Server: Add new feature state, RNG call, and prize resolution logic (including mapping of outcome ID ‚Üí client segment index).
  2. Client: Implement Bonus Wheel scene, animations, and event handling for feature_start, feature_result, and feature_end.
  3. Configuration & Math: Define wheel segment weights, prize table, and jurisdictional overrides; run simulations to validate RTP and variance.

Testing & Validation

Acceptance Criteria

  • All outcomes (coin wins, multipliers, free spins, Dragon Pot) resolve correctly and credit the player.

  • No orphaned states after disconnect / refresh.
