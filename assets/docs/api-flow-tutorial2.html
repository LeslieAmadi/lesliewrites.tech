Understanding System Communication Through APIs

A guided lesson in how systems talk to each other.

Start the Lesson
← Back to Portfolio

The Goal

In this lesson, we will send a request, observe a response, and experience how systems communicate.

Step 1 — Creating a Request

POST /risk-score
{
  "age": 52,
  "bmi": 29.4,
  "blood_pressure": 142
}

You have just created a message. This message is the beginning of system communication.

Step 2 — Observing the Response

{
  "risk_score": 0.68
}

The system has answered. You sent structure. You received meaning.

Step 3 — Adding Context

GET /risk-score/explanation
  
{
  "contributors": {
    "bmi": 40,
    "blood_pressure": 37,
    "age": 23
  }
}

Now the response becomes understandable. The system is no longer silent — it speaks.

What We Have Built

  • A request
  • A response
  • An interpretation layer
  • A meaning loop

This is system communication.

Repeat the Interaction

Change a value. Send the request again. Observe the response. Observe the explanation.

This repetition builds system intuition.
