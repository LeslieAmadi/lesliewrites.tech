ChatAssist AI — Technical Onboarding & Integration Guide

For Customer Support & Platform Teams · LexiTech

ChatAssist AI helps customer service teams respond faster and more consistently by generating draft replies directly inside your helpdesk tools. This guide walks through onboarding, connection to Zendesk/Intercom/Freshdesk, tone configuration, usage patterns, and troubleshooting. An optional API is available for advanced integrations.

Primary Users: CS managers, support agents, operations engineers
Access URL: LexiTech Dashboard
Integrations: Zendesk, Intercom, Freshdesk
Last Updated: 9 November 2025
Scope. This document assumes an organisation admin is setting up ChatAssist AI for a customer support team and may also collaborate with a technical owner for API/webhook integrations.

1. Introduction

ChatAssist AI plugs into your existing help centre and suggests draft responses for incoming tickets based on:

  • The ticket content (subject + body + conversation history).
  • Your connected knowledge base (FAQs, docs, macros).
  • Configured tone and style settings for your brand.

Agents remain in control: ChatAssist provides suggestions that can be edited, accepted, or discarded before sending.

2. System Overview & Architecture

At a high level, ChatAssist AI consists of a web dashboard, helpdesk connectors, and a suggestion engine backed by large language models and a retrieval layer over your knowledge base.

Figure 1 — ChatAssist AI Architecture (Conceptual)
Helpdesk (Zendesk / Intercom / Freshdesk)
              |
              v

              v

          |       \

The backend handles authentication, routing, knowledge base retrieval, and calls to the underlying language model. The suggestion engine is tone-aware and uses your configuration to ensure responses match your brand.

3. Requirements

To onboard ChatAssist AI for your team, you’ll need:

  • An organisation admin account in LexiTech.
  • Access to at least one supported helpdesk: Zendesk, Intercom, or Freshdesk.
  • An API key with write permissions for your organisation.
  • Modern, supported browser (Chrome, Edge, Firefox, or Safari).
  • Access to the ChatAssist dashboard via LexiTech.

4. Onboarding & Setup Flow

4.1 Step 1 — Log In

Go to the LexiTech dashboard: https://dashboard.lexitech.ai/chatassist

Sign in with your organisation admin account.

4.2 Step 2 — Connect Help Centre

Select your helpdesk under Connections:

  • Zendesk — OAuth-based connection.
  • Intercom — App install from the Intercom app store.
  • Freshdesk — API token + domain configuration.

Once connected, ChatAssist will begin indexing your help centre content (articles, FAQs, macros) to use as context when generating suggestions.

4.3 Step 3 — Configure Tone

Under Assistant Settings → Tone & Style, choose the default tone:

  • Professional – formal, concise, and neutral.
  • Friendly – warm, approachable, and empathetic.
  • Neutral – balanced and straightforward.
  • Technical – precise, detailed, and suitable for advanced users.

Tone can be overridden per-team or per-queue later, but a global default must be set during onboarding.

4.4 Step 4 — Add Team Members

Invite your team and assign appropriate roles:

  • Admin – full access: configuration, connections, and usage analytics.
  • Editor – can manage templates, examples, and tone presets.
  • Agent – can see and use ChatAssist suggestions in the helpdesk UI.

5. How ChatAssist AI Works (Agent Experience)

For agents, ChatAssist appears directly inside the ticket view of your helpdesk. It reads the conversation and suggests a draft reply the agent can review and edit.

5.1 Actions Available to Agents

For each suggestion, agents can apply quick transformations such as:

  • Shorten – make the message more concise.
  • Expand – add more detail or explanation.
  • Formalise – adjust tone to be more professional.
  • Simplify – make language clearer and easier to understand.

5.2 Typical Workflow

  1. Agent opens a ticket in Zendesk/Intercom/Freshdesk.
  2. ChatAssist reads the ticket content and knowledge base.
  3. A suggested reply appears in the ChatAssist panel.
  4. Agent edits the suggestion, optionally applies transformations (shorten, formalise, etc.).
  5. Agent sends the reply as usual.

6. Optional API & Webhook Integrations

For advanced use cases, ChatAssist can be used via API to generate suggestions outside the standard helpdesk widgets (e.g. internal tools, custom dashboards).

6.1 Example: Suggest Reply API

Endpoint (illustrative): POST /v1/chatassist/suggest-reply

Request Example

{
  "ticket_id": "ZD-48392",
  "channel": "zendesk",
  "thread": [
    {
      "from": "customer",
      "message": "Hi, my order hasn't arrived yet and it's been 10 days."
    }
  ],
  "tone": "friendly"
}

Response Example

{
  "suggestion": "Hi there, thanks for reaching out and I'm sorry for the delay. I can see that your order is still in transit. I'll contact our delivery partner to get an update and will share the latest status with you as soon as possible.",
  "tokens_used": 184,
  "model": "chatassist-support-1",
  "trace_id": "req_92cd7a01"
}

Webhooks can also be configured to log usage events or suggestion metadata back to your internal analytics systems.

7. Best Practices

  • Always review suggestions before sending – the agent remains responsible for final wording.
  • Keep your knowledge base up to date so ChatAssist has the best possible context.
  • Add example responses for complex scenarios to guide the assistant’s style.
  • Start with one queue or team, gather feedback, then roll out more broadly.

8. Troubleshooting & Error Handling

The most common issues and resolutions are listed below.

Issue Likely Cause Suggested Solution
No suggestions appearing Missing or invalid API key, or integration not fully connected. Verify connection in the dashboard and re-add the API key if needed.
Slow responses High load or temporary latency from the suggestion engine. Retry after a short delay; if persistent, contact support with affected ticket IDs.
Incorrect or unexpected tone Tone misconfigured at global or team level. Review tone settings in the dashboard and ensure the correct preset is selected.

9. Security & Data Handling

ChatAssist AI is designed to operate within typical SaaS security expectations for customer support tools.

  • All traffic between helpdesk, dashboard, and ChatAssist backend uses HTTPS.
  • Only ticket content and relevant metadata required for suggestions are processed.
  • API keys should be stored securely and rotated regularly.
