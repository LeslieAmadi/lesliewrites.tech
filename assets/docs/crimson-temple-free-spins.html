Overview

Purpose

This document defines the Free Spins Feature (Scatter Trigger) for Crimson Temple. It describes the player-facing behaviour, underlying math model, technical implementation details, and certification considerations. The goal is to provide a single source of truth for Product, Engineering, QA, Design, and Compliance.

Scope

In Scope

  • Trigger conditions for Free Spins via scatter symbols.
  • Number of awarded free spins and retrigger rules.
  • Progressive win multiplier logic during the feature.
  • Sticky Wild behaviour and symbol hierarchy within Free Spins.
  • RTP contribution and volatility impact of the Free Spins feature.
  • Front-end and back-end responsibilities for state handling.

Out of Scope

  • Base game reel strips and paytable (covered in the core Game Math Spec).
  • Jackpot or bonus wheel mechanics (handled in separate feature documents).
  • Promotional overlays (e.g. tournaments, network campaigns).
  • Operator-specific marketing copy or localisation assets (handled in Go-to-Market pack).

Prerequisites

The following must be in place before implementing or testing this feature:

  • Installed Tools: Access to the studio’s GameClient repo, GameServer repo, and Dev/QA environments.
  • Access Rights: Developer or QA access to feature flags, configuration service, and game logs.
  • Base Game Ready: Base game reels, ways evaluation, and payout engine must be stable and passing regression.
  • Math Model Signed Off: Final Free Spins math sheet approved by Head of Math and Compliance.

Dependencies

  • RNG Service: Uses the studio’s certified RNG with no feature-specific deviations.
  • Game Config Service: Multiplier growth, spins awarded, and scatter configurations are driven by environment-specific config.
  • Session & Wallet APIs: All wins generated in Free Spins are returned to the player wallet via the existing payout flow.
  • Jurisdiction Config: Minimum/maximum RTP variants per market may adjust the Free Spins hit rate or paytable weights.
  • Client Animation System: Must support symbol highlighting, multiplier animations, and sticky wild glow states.

Change Description

Current State: Base game only, 5×4, 1024 ways, standard wins and wilds. No bonus feature is currently active in the production build.

Target State: Free Spins Feature triggered by 3+ scatter symbols. The feature introduces a progressive win multiplier and sticky wild behaviour that significantly increases volatility while keeping RTP within target bounds.

Implementation Steps (High Level)

  1. Add Scatter Trigger Logic: Evaluate scatters on each base spin and award Free Spins when threshold is met.
  2. Implement Free Spins State Machine: Enter, play, retrigger, and exit sequences, including UI updates and wallet settlement.
  3. Wire Multiplier & Sticky Wilds: Increment multiplier after each winning spin; convert specific wilds into sticky positions for the remainder of the feature.
  4. Integrate with Config & RTP Variants: Allow jurisdictions to vary number of spins and multiplier growth for different RTP versions (e.g., 96.10%, 94.00%).
  5. QA, Certification, and Deployment: Run full math verification, regression, and certification submission where required.

Free Spins Mechanic — Detailed Behaviour

Trigger Conditions

  • Scatter Symbol: Golden Lotus (SCATTER) can land on any reel in both base game and Free Spins.
  • Trigger Threshold:
    • 3 scatters anywhere in view → 10 Free Spins
    • 4 scatters anywhere in view → 15 Free Spins
    • 5 scatters anywhere in view → 20 Free Spins
  • Wins from the triggering spin are paid first, then Free Spins are awarded.
  • If multiple scatter combinations occur (e.g. 3+ scatters plus high-value win), both are paid; no conflict with line/ways wins.

Free Spins Flow

  1. Base spin completes and shows 3+ scatters.
  2. Win evaluation and payout for base spin.
  3. Free Spins intro animation plays; spin counter and multiplier UI fade in.
  4. Free Spins begin with multiplier = 1x.
  5. On each winning spin:
    • Total win for that spin is multiplied by the current multiplier.
    • Multiplier then increments by +1 (e.g. 1x → 2x → 3x, etc.).
    • Any wilds that land on reels 2–4 become sticky (remain in place for remaining spins).
  6. On non-winning spins:
    • Multiplier remains unchanged.
    • Existing sticky wilds stay on the reels.
  7. Any 3+ scatters during Free Spins retrigger additional spins:
    • 3 scatters → +5 spins
    • 4 scatters → +8 spins
    • 5 scatters → +10 spins
  8. When spin counter reaches 0, final win banner is shown and all sticky wilds are cleared.
  9. Control returns to the base game with updated credit balance.

Game Math & RTP Contribution

  • Target overall game RTP: 96.10% (base RTP variant).
  • Approx. Free Spins contribution to RTP: 22–26% depending on jurisdiction config.
  • Hit frequency for Free Spins (3+ scatters): ~1 in 160–210 spins (configurable by reel strip tuning).
  • Feature is designed as the primary high-volatility component of the game.

  • Multiplier behaviour matches specification (increments only after wins, persists between spins).

  • All supported jurisdictions pass compliance and certification checks.
