Reference API Explainability Interpretability Service

Explainability API

Authoritative service contract for model interpretability and prediction transparency.

← Back to Portfolio
On this page: Endpoint · Request Parameters · Response Schema · Error Model · Governance Properties

Endpoint

GET /risk-score/explain

Returns interpretability data for the most recent prediction request.

Request Parameters

{
  "prediction_id": string
}

The prediction identifier links explanation output to a specific inference event.

Response Schema

{
  "contributors": {
    "bmi": percentage,
    "bp": percentage,
    "age": percentage
  }
}

Each contributor represents relative influence on the prediction outcome.

Error Model

{
  "error_code": string,
  "message": string,
  "trace_id": string
}

Errors are traceable, auditable, and linked to inference events.

Governance Properties

  • Traceability of predictions
  • Explainability audit trails
  • Interpretability guarantees
  • Model transparency controls
  • Human-review enablement

This service functions as a trust layer between automation and human decision-making.
